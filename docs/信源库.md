# 信源库

## 使用哲学

信源库不是名单墙，而是 agent 的导航图。每次生成早报时，要先判断信息源的角色：

- **事实源**：官方、考试院、学校官网/公众号、赛事组委会。正文事实优先引用这里。
- **线索源**：媒体、升学公众号、第三方账号目录、内部线索。用于发现选题，不能直接证明事实。
- **解释源**：有经验的升学号、家长观察、内部咨询反馈。用于理解家长关心什么，但要回到事实源核验。

抓取时优先“能稳定拿到原始事实”的路径：官网列表页、栏目页、PDF、官方公众号原文、考试院/教委通知。微信公众号、JS 站、报名系统不稳定时，用 `web-access` 的浏览器 CDP、完整文章 URL、截图、PDF 附件和人工关注兜底。

正文不要写“扫描了哪些源、剔除了哪些源”；这些放在信源扫描记录里。

## 访问方法分级

| 方法 | 适用场景 | 注意事项 |
|---|---|---|
| 普通网页抓取 | 教委、考试院、学校官网、新闻列表 | 优先方式，保留标题、日期、URL |
| 搜索引擎 + 站内限定 | 栏目层级深、官网搜索弱 | 用 `site:域名 关键词` 定位原文 |
| Jina / 页面转 Markdown | 正文型网页、政策文件、新闻稿 | 可省 token，但重要事实仍保留原 URL |
| PDF 下载/文本抽取 | 招生方案、规则、名单、公示 | 文件名、发布日期、发布单位要记入扫描记录 |
| 浏览器 CDP | 微信文章、JS 站、综评网、报名页、证书页 | 创建后台 tab；保留完整 URL；必要时截图 |
| 微信内搜/搜狗微信 | 公众号历史文章发现 | 只作定位线索，需回官方或认证主体核验 |
| 人工兜底 | 小程序、登录系统、验证码、公众号历史消息 | 记录需人工打开的入口和原因 |

## 上海各区青少年活动中心、少年宫、少科站

这类信源必须纳入科创升学扫描。原因：很多区级青少年科技创新大赛、明日科技之星、雏鹰杯、AI/机器人竞赛、科学营、综评相关实践项目，不一定先出现在学校官网，常先由区青少年活动中心、少年宫、少科站或区教育局相关栏目发布。

### 发布位置规律

| 发布位置 | 常见内容 | 抓取方式 | 判断 |
|---|---|---|---|
| 区青少年活动中心/少科站官网 | 竞赛通知、报名表、获奖名单、科技创新栏目、活动快讯 | 普通网页；栏目页轮询；PDF 附件下载 | 最优先，尤其有“科技创新/竞赛通知/通知公告”栏目时 |
| 区教育局政务公开/通知公告 | 正式文件、区级比赛通知、科技节通知、项目名单 | 普通网页/PDF；站内限定搜索 | 事实源，适合引用正文 |
| 区政府新闻/暑期项目表 | 活动报道、暑期项目、报名渠道、公众号说明 | 普通网页/PDF | 线索源或补充事实 |
| 公众号/报名系统 | 面试安排、报名入口、临时通知、入围名单 | 微信内搜、搜狗微信、CDP、人工关注 | 重要但不稳定，需保存完整 URL/截图 |

优先关键词：`青少年活动中心`、`少年宫`、`少科站`、`少年科技指导站`、`科技创新大赛`、`明日科技之星`、`科技节`、`雏鹰杯`、`创意设计`、`人工智能`、`机器人`、`科学营`、`科学研究院`、`创新人才`、`报名`、`获奖名单`、`公示`。

### 抽检后形成的判断

不要只扫“区青少年活动中心首页”。不同区的有效发布位置差异很大：

- **中心官网型**：徐汇、黄浦、宝山、松江、青浦这类入口较清楚，重点轮询“科技创新/活动竞赛/竞赛通知/竞赛成绩/通知公告/区少科站”。
- **教育局文件型**：浦东、嘉定、长宁、杨浦这类经常由区教育局或政务公开发布正式通知，活动中心/少科站是承办或办公室关键词。
- **公众号补充型**：虹口、闵行、奉贤等区，报名入口、面试安排、临时通知可能落在区青少年活动中心公众号；正文引用前仍要回政府、教育局、中心官网或官方认证号核验。
- **新闻线索型**：静安、普陀、金山、崇明等区，有时先在区政府新闻、上海发布/上观区号、上海科普网出现活动报道；这类内容优先作为线索，除非能追到正式通知或主办单位原文。

5. 搜狗微信可能验证码或缺页，只能用于定位，不能当事实证明。
